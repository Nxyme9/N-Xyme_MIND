## What I am

Category gateway for **plan** workflows.
I organize multiple sub-workflows so you pick only what you need.

## Sub-workflows

- **`bmad-create-product-brief`** — Guided discovery to define product, audience, and success criteria
- **`bmad-product-brief-preview`** — Autonomous product brief creation or update
- **`bmad-edit-prd`** — Modify existing product requirements document
- **`bmad-validate-prd`** — Check PRD against quality standards
- **`bmad-create-ux-design`** — Plan UX patterns and design specifications
- **`bmad-create-architecture`** — Solution design decisions and tradeoffs
- **`bmad-create-epics-and-stories`** — Break requirements into epics and user stories
- **`bmad-check-implementation-readiness`** — Validate all specs are complete before build

## How to use

1. Review the sub-workflows above
2. Identify which one matches the user's need
3. The full workflow instructions are in `bmad/` — read the relevant `workflow.md` file:
      - `bmad/.../bmad-create-product-brief/workflow.md`
   - `bmad/.../bmad-product-brief-preview/workflow.md`
   - `bmad/.../bmad-edit-prd/workflow.md`
   - `bmad/.../bmad-validate-prd/workflow.md`
   - `bmad/.../bmad-create-ux-design/workflow.md`
   - `bmad/.../bmad-create-architecture/workflow.md`
   - `bmad/.../bmad-create-epics-and-stories/workflow.md`
   - `bmad/.../bmad-check-implementation-readiness/workflow.md`
4. Follow the steps in that file
