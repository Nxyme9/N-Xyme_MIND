## What I am

Category gateway for **build** workflows.
I organize multiple sub-workflows so you pick only what you need.

## Sub-workflows

- **`bmad-create-story`** — Generate a story spec file with full context
- **`bmad-dev-story`** — Execute story implementation from spec
- **`bmad-quick-spec`** — Quick tech spec for small changes
- **`bmad-quick-dev`** — Implement quick tech spec
- **`bmad-quick-dev-new-preview`** — Implement any change request
- **`bmad-correct-course`** — Manage sprint changes

## How to use

1. Review the sub-workflows above
2. Identify which one matches the user's need
3. The full workflow instructions are in `bmad/` — read the relevant `workflow.md` file:
      - `bmad/.../bmad-create-story/workflow.md`
   - `bmad/.../bmad-dev-story/workflow.md`
   - `bmad/.../bmad-quick-spec/workflow.md`
   - `bmad/.../bmad-quick-dev/workflow.md`
   - `bmad/.../bmad-quick-dev-new-preview/workflow.md`
   - `bmad/.../bmad-correct-course/workflow.md`
4. Follow the steps in that file
