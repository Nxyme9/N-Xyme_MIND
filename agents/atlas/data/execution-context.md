# Atlas — Execution Context

## Plan Format (Prometheus Output)
Plans generated by Prometheus follow this structure:
- `## Tasks` — bullet list of `- [ ]` checkbox items
- Each task has: description, target agent, verification criteria
- Dependencies indicated by ordering (earlier = prerequisite)
- `## Verification` — final quality checks

## Tracking Key Convention
Memory keys follow this pattern for consistency:

| Key | Value | Example |
|-----|-------|---------|
| `atlas:plan:{plan_name}:status` | `pending|in_progress|complete|failed` | `atlas:plan:refactor-auth:status` |
| `atlas:plan:{plan_name}:tasks` | JSON array of tasks | `atlas:plan:refactor-auth:tasks` |
| `atlas:plan:{plan_name}:task:{id}` | `pending|in_progress|complete|failed|blocked` | `atlas:plan:refactor-auth:task:1` |
| `atlas:plan:{plan_name}:report` | Markdown progress report | `atlas:plan:refactor-auth:report` |

## Status State Machine
```
pending → in_progress → complete
                      → failed → in_progress (retry)
                      → blocked (escalated to user)
```

## Reporting Templates

### Progress Update (after each task)
```
📊 Progress: {X}/{N} tasks complete
✅ {task_name} — done and verified
⏳ {task_name} — in progress
❌ {task_name} — blocked: {reason}
```

### Completion Summary
```
✅ Plan Complete: {plan_name}
Tasks: {complete}/{total}
Blockers: {count}
Duration: {estimated}
Outstanding: {issues or "none"}
```

### Blocker Alert
```
🚫 BLOCKER: {task_name}
Reason: {what went wrong}
Attempts: {retry_count}
Recommended: {suggested resolution}
```

## Delegation Quick Reference
| Tool | Blocks? | Identity? | When |
|------|---------|-----------|------|
| `delegate_task` | Yes | Full | Need result before continuing (serial) |
| `call_omo_agent` | No | Full | Parallel independent tasks |

## Anti-Hallucination
- Read `data/anti-hallucination-rules.md` at startup
- Always verify tool names against MCP server definitions
- Never assume a file exists without globbing or reading it first
- Cross-check every subagent claim against actual output
